# 授信审批部内部工作流方案

> 起草日期：2026年4月23日
> 适用范围：分行授信审批部，6科室全体

---

## 一、部门职责与科室分工

### 1.1 部门核心职能
信贷审查审批、放款管理、信贷档案归档。

### 1.2 科室分工

| 科室编号 | 科室名称 | 主责业务类型 | 科室评审SLA |
|----------|----------|-------------|------------|
| D1 | 地产基建评审科 | 房地产开发贷款、城建基建贷款 | ≤3工作日 |
| D2 | 产业评审科 | 制造业、商贸服务业、科技型企业等产业类贷款 | ≤3工作日 |
| D3 | 跨境评审科 | 跨境融资、国际结算融资、进出口信货、外汇政策相关 | ≤5工作日 |
| D4 | 投行金融市场评审科 | 债券投资、ABS、北金所理财直融、金融市场创新业务 | ≤5工作日 |
| D5 | 普惠业务评审科 | 小微企业贷款（含税贷）、个人经营贷、消费贷 | ≤2工作日 |
| D6 | 放款科 | 放款条件落实、合同用印、台账登账、档案归档 | ≤1工作日 |

> 注：D6放款科为流程终端科室，不参与审查环节，按序时接单。

## 二、信贷业务流程总图

```
客户申请
    │
[受理] ─── D1/D2/D3/D4/D5 ─── 责任人：各科室受理岗
    │ （1工作日内完成受理登记）
    ▼
[初审] ─── 责任人：科室主办信贷员
    │ （1工作日内完成材料完整性初核）
    ▼
[科室评审] ─── 责任人：科室负责人
    │ （D1/D2≤3日，D3/D4≤5日，D5≤2日）
    ▼
[部级评审] ─── 仅以下情形触发：
    │ ① 单笔金额≥分行授权上限
    │ ② 涉及担保圈、关联互保
    │ ③ 跨境业务涉及外汇管制
    │ ④ 投行类创新业务
    │ ⑤ 主任授信审批人认为需部级审议的
    ▼
[审批批复] ─── 责任人：有权审批人
    │ （批复有效期30日，特殊可申请延长15日）
    ▼
[放款落实] ─── D6放款科
    │ （收到完整放款材料后≤1工作日）
    ▼
[归档确认] ─── D6放款科档案岗
    │ （放款后5个工作日内完成档案整理归档）
    ▼
结案
```

## 三、时效节点与预警机制

### 3.1 节点时限总表

| 节点 | 时限要求 | 责任方 | 触发条件 |
|------|---------|-------|---------|
| 受理 | 1工作日内 | 各科室受理岗 | 业务分配后 |
| 初审 | 1工作日内 | 主办信贷员 | 受理完成后 |
| 科室评审 | 见科室SLA表 | 科室负责人 | 初审通过后 |
| 部级评审 | 2工作日内 | 部级贷审委 | 科室上报后 |
| 审批批复 | 2工作日内 | 有权审批人 | 部级通过后 |
| 放款落实 | 1工作日内 | 放款科 | 完整放款材料到位 |
| 归档 | 5工作日内 | 放款科档案岗 | 放款完成后 |

### 3.2 预警等级

| 等级 | 触发条件 | 通知范围 | 处置要求 |
|------|---------|---------|---------|
| 🔴 A级-红灯 | 超时未完成 | 科室负责人→部门负责人 | 立即整改，24h内说明原因 |
| 🟡 B级-黄灯 | 剩余≤1工作日 | 科室负责人 | 当日跟进，当日反馈 |
| 🟢 C级-绿灯 | 剩余≥2工作日 | 正常 | 按计划推进 |

> 超时业务冻结科室绩效考核评分，直至整改完毕。

## 四、责任矩阵（RACI）

| 流程节点 | 主办信贷员 | 科室负责人 | 放款科 | 有权审批人 | 部门负责人 |
|---------|-----------|----------|--------|-----------|----------|
| 受理 | R | I | - | - | - |
| 初审 | R | A | - | - | - |
| 科室评审 | C | R | - | I | I |
| 部级评审 | C | R | - | A | A |
| 审批批复 | I | I | - | R | A |
| 放款落实 | I | I | R | - | - |
| 归档确认 | - | - | R | - | - |

> R=主办 A=审批 I=知会 C=协助

## 五、日常管理机制

### 5.1 日管控
- **每日9:00前**：台账系统自动导出当日预警清单（A级+B级），科室负责人确认
- **每日17:00前**：各科室更新在办业务节点状态，无进展须备注原因

### 5.2 周通报
- **每周一**：部门工作台账汇总，部门负责人在周会上通报
- 周报内容：本周新接业务量、各科室在办量、超时/预警数量、环比变化

### 5.3 月度检视
- 每月末由放款科汇总全月业务数据（含金额、业务类型、户数）
- 超时业务逐一登记原因，纳入科室月度考核

## 六、档案管理规范

| 归档节点 | 归档内容 | 时限要求 | 存放位置 |
|---------|---------|---------|---------|
| 归档确认 | 批复、合同、放款凭证、抵质押材料全套 | 放款后5工作日内 | 档案中心 |
| 档案借阅 | 须登记借阅事由，原则上不外借原件 | - | - |
| 档案销毁 | 贷款结清后保存15年，特殊情况按监管要求 | 结清后第16年起 | 按规定流程 |

## 八、台账系统使用说明

### 8.1 系统位置
```
/Users/bismarck/KnowledgeBase/CreditWiki/scripts/dept_workflow.py
```

### 8.2 常用命令

```bash
# 查看在办业务台账（按紧急程度排序）
python3 dept_workflow.py list

# 按科室查看
python3 dept_workflow.py list --dept D3

# 新增一笔业务
python3 dept_workflow.py add D5 某科技公司 500 科技贷款 科室评审 张三

# 推进节点
python3 dept_workflow.py advance 20250423102030 审批批复 批复已下达

# 查看业务详情
python3 dept_workflow.py detail 20250423102030

# 导出预警报告
python3 dept_workflow.py export

# 科室统计
python3 dept_workflow.py dept

# 全量统计
python3 dept_workflow.py stats
```
